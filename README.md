# BLB安全リネームツール（BLB Safe Renamer）

## 📌 これは何をするツールですか？

**たくさんのファイルの名前を、一覧表のとおりに一括で変更するツール**です。

たとえば「`f340.blb` というファイルを `Honbun2.txt` という名前に変えたい」という組み合わせが何千件もあるとき、1件ずつ手作業で変更するのは大変ですし、間違いも起きます。このツールを使うと、対応表（`rename_map.txt`）に書いたとおりに、まとめて自動で変更できます。

### 安心して使えるポイント

- ✅ **「お試し確認」と「本番実行」が分かれています**
  いきなりファイル名が変わることはありません。まず「お試し確認」で、どう変わる予定かをチェックできます。
- ✅ **同じ名前がぶつかっても大丈夫**
  変更後の名前が重なった場合は、自動で `_002`、`_003` と番号を付けて区別します。
- ✅ **結果がすべて記録に残ります**
  何がどう変わったか（変わらなかったか）が、Excel で開ける一覧表（CSV）に保存されます。
- ✅ **ファイルの中身には一切触りません**
  名前を変えるだけで、ファイルの中身を書き換えたり削除したりすることはありません。

## 🖥️ 必要なもの

| 項目 | 内容 |
|---|---|
| パソコン | Windows のパソコン（Windows 10 / 11 なら追加インストール不要） |
| 権限 | 管理者として実行できること |
| 特別なソフト | 不要（Windows に最初から入っている機能だけで動きます） |

## 📁 フォルダの中身

```text
BLB-Safe-Renamer/
├─ input/               ← 📥 名前を変えたいファイル（f340.blb など）をここに入れます
├─ rename_map.txt       ← 対応表（あなたが編集するファイル）
├─ run_precheck.bat     ← ① お試し確認用（ファイルは変わりません）
├─ run_execute.bat      ← ② 本番実行用（実際に名前が変わります）
├─ Rename-BlbSafe.ps1   ← ツール本体（編集不要・触らないでください）
├─ output/              ← 予備フォルダ
└─ logs/                ← 結果の記録（CSV）が自動で保存されます
```

**名前を変えたいファイルは、すべて `input` フォルダの中に入れてください。**
（`input` フォルダがない場合は、一度実行すると自動で作られます）

## 🚀 使い方（この順番を必ず守ってください）

```text
1️⃣ 名前を変えたいファイルを input フォルダに入れる
2️⃣ バックアップを取る（input フォルダごとコピーしておく）
3️⃣ 対応表（rename_map.txt）を書く
4️⃣ お試し確認（run_precheck.bat）を実行する
5️⃣ 記録（CSV）を見て、予定どおりか確認する
6️⃣ 本番実行（run_execute.bat）を実行する
7️⃣ 記録（CSV）を見て、結果を確認・保管する
```

### 手順1️⃣ ファイルを input フォルダに入れる

名前を変えたいファイル（`.blb` ファイルなど）を、すべて `input` フォルダの中にコピーまたは移動してください。

### 手順2️⃣ バックアップを取る

万一に備えて、**input フォルダごとコピーして別の場所に保存**しておいてください。件数が多いほどやり直しが大変なので、この手順は省略しないでください。

### 手順3️⃣ 対応表を書く（rename_map.txt）

メモ帳などで `rename_map.txt` を開き、1行に1件ずつ、次の形で書きます。

```text
ren "元のファイル名" "新しいファイル名"
```

書き方の例：

```text
ren "f340.blb" "Honbun2.txt"
ren "f341.blb" "Honbun3.txt"
ren "f30715.blb" "f30715.pdf"
```

📝 書き方のルール：

- 半角の `ren`、半角スペース、ダブルクォート（`"`）を必ず付けてください
- `rem` または `::` で始まる行はメモ（コメント）として無視されます
- 空の行があっても問題ありません
- 拡張子（`.txt` や `.pdf` の部分）を変えることもできます

### 手順4️⃣ お試し確認（run_precheck.bat）

1. `run_precheck.bat` を**右クリック**
2. 「**管理者として実行**」を選ぶ
3. 黒い画面に結果のまとめが表示され、`logs` フォルダに記録（CSV）が保存されます

⚠️ この段階では**ファイル名は1つも変わりません**。安心して実行してください。

### 手順5️⃣ 記録を確認する

`logs` フォルダの中の一番新しい CSV ファイルをダブルクリックすると、Excel で開けます。

特に見てほしい列は次の2つです。

- **Status 列** … 各行の結果（下の「記録の見方」を参照）
- **TargetActual 列** … 実際に付く予定の名前（番号付きになる場合はここで分かります）

### 手順6️⃣ 本番実行（run_execute.bat）

お試し確認の内容に問題がなければ、`run_execute.bat` を**右クリック → 「管理者として実行」**。今度は実際にファイル名が変わります。

### 手順7️⃣ 結果を確認・保管する

もう一度 `logs` フォルダの最新の CSV を開き、結果を確認してください。この CSV は作業の証拠になるので、削除せずに保管することをおすすめします。

## 📊 記録（CSV）の見方

| 列の名前 | 意味 |
|---|---|
| Time | 処理した日時 |
| Mode | DRYRUN＝お試し確認 / EXECUTE＝本番実行 |
| LineNo | 対応表の何行目か |
| Status | 結果（下の表を参照） |
| Source | 元のファイル名 |
| TargetRequested | 対応表に書いた新しい名前 |
| TargetActual | 実際に付いた（付く予定の）名前 |
| Reason | 補足の説明 |
| Error | エラーの内容（問題があった場合のみ） |

### Status（結果）の意味

| Status | 意味 | 対応 |
|---|---|---|
| DRYRUN_OK | お試し確認で問題なし ✅ | そのまま本番実行へ |
| OK | 本番実行で名前の変更に成功 ✅ | 対応不要 |
| SKIP_PARSE | 対応表の書き方が正しくない ⚠️ | その行の書き方を直してください |
| SKIP_SOURCE_MISSING | 元のファイルが見つからない ⚠️ | ファイル名の綴りを確認してください |
| SKIP_INVALID_TARGET | 新しい名前に使えない文字がある ⚠️ | 名前を直してください（下の注意事項を参照） |
| SKIP_DUPLICATE_SOURCE | 同じ元ファイルが対応表に2回以上ある ⚠️ | 2回目以降は無視されます。対応表を確認してください |
| ERROR_RENAME_FAILED | 名前の変更に失敗した ❌ | Error 列を確認。ファイルを開いたまま実行していないか確認してください |

### 名前が重なったときの自動採番

同じ「新しい名前」が2件以上あると、2件目からは自動で番号が付きます。

```text
Honbun2.txt        ← 1件目
Honbun2_002.txt    ← 2件目
Honbun2_003.txt    ← 3件目
```

フォルダ内にすでに同じ名前のファイルがある場合も、同じように番号が付きます（既存のファイルが上書きされることはありません）。重なりが1,000件を超えると `_1000` のように番号の桁が増えます。

## ⚠️ 注意事項

- 🔒 **本番実行の前に、必ずバックアップとお試し確認を行ってください**
- 📁 名前を変えられるのは、**input フォルダに直接入っているファイルだけ**です（input の中にさらにフォルダを作っても、その中は対象外）
- 🚫 新しい名前に次の記号は使えません： `< > : " / \ | ? *`
- 🚫 名前の最後をスペースやピリオド（`.`）にすることはできません
- 🚫 Windows が予約している特別な名前（CON、PRN、AUX、NUL、COM1〜COM9、LPT1〜LPT9）は使えません
- ℹ️ このツールはファイルの中身を確認しません。対応表に書いた名前が正しいかどうかは、作成者が責任を持って確認してください

## ❓ よくある質問・トラブル

| 症状 | 原因と対処 |
|---|---|
| 黒い画面に `ERROR: rename map not found` と出る | `rename_map.txt` がツールと同じフォルダにありません。置き場所を確認してください |
| SKIP_SOURCE_MISSING がたくさん出る | ファイルが input フォルダに入っていないか、元のファイル名の綴り・拡張子が実際のファイルと合っていません |
| SKIP_PARSE が出る | その行が `ren "元の名前" "新しい名前"` の形になっていません。ダブルクォートの付け忘れが多いです |
| SKIP_INVALID_TARGET が出る | 新しい名前に使えない文字（上の注意事項参照）が含まれています |
| ERROR_RENAME_FAILED が出る | そのファイルを Excel などで開いたままになっていませんか？ 閉じてからもう一度実行してください |
| 実行しても一瞬で画面が消える | バットファイルをダブルクリックではなく、**右クリック →「管理者として実行」**で起動してください |
| 大量のファイルだと時間がかかる | 28,000件でおよそ10分程度かかります。黒い画面が出ている間はそのままお待ちください |
